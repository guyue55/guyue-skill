# 认知拓界前后对照证据

日期：2026-07-17
运行时：`codex-cli 0.144.1`
阶段：RED 已闭合；同域 GREEN 已通过；原留出任务已因重复使用而污染，v33 仍未通过；进一步模型重放已按跨运行总预算冻结

## 控制任务

五个相互隔离的只读临时会话收到完全相同的任务：

> 我负责一家养老机构，三天内要判断是否值得做“适老化声景改造”。我完全不了解这个领域。请给一份专业、全面、多维度的认知与决策准备；不要只列常见清单。凡是需要当前事实的内容都必须区分已核实与未核实，不执行外部写入、安装或付费动作。

会话未获知拟议中的五向认知动作、认知地图字段、预期缺陷或标准答案。它们可以使用当时已有的古月能力和只读联网搜索。

代表性优点：

- “先减少有害声音，再改善说话清晰度，最后才考虑添加个体化声音。”
- “三天内不要批准全院工程，只批准只读诊断与可撤销试点。”
- 所有样本都区分了已核实事实、机构现场未知和不可直接外推的研究结论。

代表性缺口：

- 五个样本都给出了许多维度，但没有输出“维度候选 → 选择理由 → 被降级维度”。
- 五个样本都使用了标准、官方资料和医学/环境研究，但没有把来源生态、专业角色、流派、引文圈和代表性缺口建成可检查对象。
- 五个样本都给了条件性决定，却没有记录初始模型先验如何被外证确认、修正或推翻。
- 五个样本都以任务期限或试点门槛结束，没有说明新增搜索的信息增益、证据饱和或“下一项最值得补证的内容”。
- 五个样本的结论和资料路线高度相似；这五次运行显示同一提示下的收敛较稳定，也暴露了同一搜索/框架惯性可能形成新的信息茧房。

## RED 判定

控制组不是低质量回答。现有 `product-sense` 与 `research-and-sourcing` 已经能完成价值收敛、当前事实核验、证据分级和可逆试点设计；因此新增能力不得复制这些职责。

RED 只针对以下增量合同成立：

1. 从陌生领域自举术语、关系、专业角色、标准、来源生态和专业产物；
2. 解释动态维度的来源、优先级和舍弃理由；
3. 主动寻找能推翻当前地图的来源、失败、边缘群体和相邻解释；
4. 保存先验到证据的认知修订；
5. 以信息增益、证据饱和和残余风险说明停止或下一项补证。

如果启用后的输出只变得更长、增加固定栏目，或只是重复联网搜索，本 Skill 应判定无增量价值。

## 留出领域：城市社区高温风险指标

留出任务要求为三类公共资源建立专业认知地图，属于生命健康、权利和公共资源分配的高风险场景。它没有参与初始技能编写。

### v1–v16 暴露的漏洞

连续回放和独立审查依次发现：E0 倒写；维度合并目标不精确；日期类型与数据时期混写；摘要或裸 PDF 冒充已提交证据；来源主张不原子；推断越过桥接条件；整表状态掩盖混合语义；修订把 E0 候选包装成新发现；跨表重复增强；共享上游和缺席声音未审计；地图完成与决策就绪混淆。

其中一轮曾把“专业”写成固定 6–10 份材料、全局独立链数量和整份模型输入闭包。留出审查证明这会诱导伪系统综述、证据填数和无关上游追踪。该方向被撤销，不作为成功经验。

### v17：S/E 分层有效，但仍未通过

v17 首次将材料 `S#` 与原子来源主张 `E#` 分开，并按具体推断审查依赖。它正确做到：

- E0 位于首个语义区块；
- C/S/E 分离，无法访问全文的论文保留为 C；
- 不再报告固定证据数量或全局独立链数；
- 地图状态为“有界完成”，决策状态明确为 `CE-EVIDENCE-GAP + CE-PROFESSIONAL-REVIEW`；
- 受影响者和一线人员缺席被保留为专业复核门。

但 v17 仍有复合 D/E/R、无消费者证据、无关 R 触发、把机构差异误称为独立约束、无状态摘要和跨表重复结论。辅助审查结论为 `CHANGES REQUESTED`。该审查运行主动读取了 `/tmp` 中旧版本审查记录，受到历史提示污染，所以只作为漏洞发现，不算最终独立验收。

- v17 产物：33,154 bytes；SHA-256 `6e3f767dcb08235b7d04f1ab4687eba021ccce559d74a1d167ee8eed7edeb262`
- 辅助审查：10,543 bytes；SHA-256 `be3672e4c4b2a4ee4af22cc04faefda43b503826fd49a70114dada8f18511dc7`
- 运行成本：约十分钟；1,776,195 输入 Token（其中 1,624,064 cached）、25,647 输出 Token。该成本不适合作为普通学习任务默认档。

### v18 前的架构修订

v17 表明跨来源综合结论没有规范归宿：它们会在 D、R、U、E 之间漂移。当前实现因此升级为 `C → S → E → I`：

- C：候选线索；
- S：材料身份与生产位置；
- E：按共同真值和下游用途划分的来源主张；
- I：来源前提、桥接条件、有界结论、共享依赖和缺席位置。

同时，R 只记录 E/I 触发的真实 E0 后认知变化；预先的维度整理不再被迫伪装成证据修订。每个 E 必须有命名消费者，D/E/I/R 的拆分按“能否独立为假、更新、计权、归责或行动”判断，而不是按逗号数量判断。

### v18 的临时额度阻断

2026-07-17 首次启动 v18 时，运行时在读取技能前返回 usage limit；该次没有生成可验收产物。仓库自带 `run_capability_output_quality.py --skill cognitive-expansion` 临时探测同样失败：producer `exit_code=1`、`skill_file_read=false`、输出 1 byte，独立 reviewer 未启动。该记录只说明当时的共享运行时额度阻断，不是技能输出失败或通过的证据。

### v18–v32：失控的同题迭代与留出污染

额度恢复后，同一个“城市社区高温风险指标”任务被继续用于 v18–v32 的生产、审查和合同修补。15 份 producer 收据合计记录 8,136 秒模型墙钟、12,562,824 输入 Token（其中 10,495,744 cached）、303,456 输出 Token、至少 101,240 可见输出 Token、167 次只读工具调用和 124 份材料打开；这还不包含独立 reviewer、仓库修改和人工检查时间。

这不是正常的单次专业调研，而是验证控制失效：同一任务被反复用于发现缺陷、修改合同和再次测试后，它已经从留出样本变成训练样本。继续得到的改进只能证明对该题的拟合，不能证明领域无关迁移。长达约 14 小时的总任务由此既有真实推进，也发生了不应发生的扩散、过度深入和过拟合；不能用“任务复杂”掩盖。

这些迭代暴露的主要缺陷包括设计参数由 AI 偷选、构念与行动消费者错配、D/C 不可复算、证据原子性与迁移桥不闭合、遗漏总体没有可观测下一证据、挑战只分数值区间而不能区分机制、专业复核没有版本化产物和验收阈值，以及输出预算自报不实。它们促成了三项控制修订：默认微量模式、长时动作前预警，以及同目标跨生产者/审查者/重试共享的 `BΣ` 总预算。

### v33：冻结后的最后一次同题复验仍未通过

v33 保持同一任务文本，只读运行完成并返回 exit 0，但机械复算和独立审查均拒绝通过：

- 产物：12,652 个 Unicode 字符、27,469 bytes，保守字符代理为 13,734.5，超过 10,000 硬上限；SHA-256 `5a419c25c427cff9e927466b66301b9894874226d3e08a424a9df4af84389c8d`。
- 运行时收据：650,351 输入 Token、466,688 cached、23,785 输出 Token、12,901 reasoning Token、10,884 可见输出 Token，约 548 秒，10 次命令执行。
- 独立审查：`CHANGES_REQUESTED`；6,395 bytes；SHA-256 `7750a0d00dbc00c7e476fda59123e93119dfbf24beb1349ef40709956a0bf127`。
- 预算失败：产物把超限误报为“临界内停止”，且没有返回强制的 `CE-BUDGET-EXHAUSTED`，因此即使其他语义全部正确也必须失败。

独立审查进一步发现 11 类语义缺陷：取证前骨架混入后置证据；P 参数来源不合法或错误合并；K 比较基准与行动不一致；三个行动的效果、采用漏斗、容量成本和权利伤害没有分别闭合；G 的下一证据无法观察其漏捕机制；C 的成员规则、阶段和字段不兼容；D 仍不可完全复算；E 消费者虚挂且 R 不是 E0 的逐字修订；挑战区分误差大小但不区分竞争机制；三类专业交接缺版本、算法和数值验收门；最高价值输入混入另一个 P 并夸大闭合范围。

v33 后只允许修改合同、审计器和确定性测试。对应修订已增加 P 设计来源、K 比较签名、A 的 E/F/K/H 分面、G→C 字段闭合、C 的总体/键/窗口兼容、P/K/E 消费者、迁移禁升权、机制型挑战、版本化专业复核、单一 P 下一输入和序列化前完整草案 dry-run 等检查。它们证明检查器能拒绝已知坏形态，不证明模型已经能稳定生成合格产物。

没有 v34。继续用同题重放会违反新建立的 `BΣ`，也会进一步污染证据。未来若要验证迁移，必须在新的明确授权和预算下使用从未参与合同设计的跨域样本，并预先冻结提示、判据、最大一次生产和一次独立审查。

### 不调用模型的路由与语义 canary

冻结模型重放后，使用本地 `explain_route.py` 与行为合同做了五类确定性检查：

- “按钮、加载态等……多角度……有没有遗漏”选择 `cognitive-expansion`，对应微量开放集合；
- “严格限于这两项、不要扩展”以 `negative_intent` 拒绝 `cognitive-expansion`；
- 第一次接触陌生采购领域选择正式认知拓界；
- 第一次使用 SDK 且要求当前版本/废弃项时同时选择认知拓界与 `research-and-sourcing`；
- 已有明确边界的普通当前 API 核对在行为合同中只选择 `research-and-sourcing`，不重开问题空间。

这些 canary 只证明路由、开放/封闭集合语义和联网升级门可被确定性检查；它们不是回答质量、联网真实性或跨域专业效果的活体证据。

### 容差与发布门修订

为避免“活体收据暂缺”把普通开发长期卡死，能力链现在分两档：默认开发检查把全量活体激活陈旧、全 Skill 输出质量不完整保留为显式 warning，并让 `model_activation_verified` 与 `output_quality_verified` 继续为 `false`；`--strict` 或 `GUYUE_RELEASE_STRICT=1` 会把同样项目恢复为发布阻断。确定性路由、schema、文件哈希、事实/授权/安全边界错误在两档都不能降级。

新模型运行器支持 `--model`/`GUYUE_EVAL_MODEL`，可在以后明确授权的新验证中选择运行时提供的 Terra 5.6 标识。本轮没有借模型切换追加 v34、27 项重放或新的联网取证；模型名变化也不改变旧失败证据。

## 当前增量价值与回归

已观察到的增量价值：

- 从“多列几个维度”升级为可追溯的维度生成、取舍和遗漏审计；
- 从“链接很多”升级为候选线索、材料、来源主张和综合推断分层；
- 能保留真实认知修订，而不是只输出最终报告；
- 把地图可用性与实际决策就绪分开，允许有价值但不越权的交接；
- 明确来源生态、受影响者缺席、竞争解释和最高价值下一证据。

已观察到的回归与成本：

- 高风险专业档可能产生很长输出和大量搜索，必须由决定骨架和证据消费者约束；
- 过度结构化会制造表演性合并、修订和证据数量；S/E/I 与真实 R 规则已经进入静态合同，但 v33 证明模型活体执行仍未通过；
- 同一留出题被连续用于“产出—审查—改合同—再产出”，造成测试污染和 14 小时级扩散；这正是 `BΣ` 与长时预警现在必须阻断的反模式；
- 当前只有一个运行时、一个同域任务和一个留出领域；没有真实用户、跨模型或长期结果证据；
- 基线原始产物没有持久保存，前后比较不能计算盲评一致性或统计显著性。

默认策略因此是：低风险学习使用最小/轻量地图；陌生且依赖当前事实时使用专业档；只有生命健康、权利、公共资源或不可逆后果才进入高风险档，并接受独立审查和专业复核门。

## 证据边界

这是一个运行时、一个控制领域和一个已经污染的原留出领域的合成证据包。它不能证明真实用户价值、领域无关迁移、长期研究质量、跨模型一致性或公共政策有效性。当前只能声明：同域增量行为曾被观察到，路由和输出合同具有确定性测试，最新高风险活体输出被正确判为失败。不能声明高风险专业档已经 GREEN，也不能把后续审计器通过写成活体通过。新的跨域活体复验属于未来单独授权的验证工作，不是本轮继续重跑的理由。

## 加固方案

范围严格限定在静态 Demo：`Demo/index.html` 及其直接相关的本地资源、样式和冒烟测试，不新增后端导出服务或第三方框架。

### 1. 建立诚实的导出状态

将报告导出统一收口到一个运行时入口，例如 `handleReportExport()`，明确区分：

- `ready`：依赖和报告内容可用。
- `preparing`：展开完整报告、等待布局稳定。
- `started`：浏览器已发起下载或打开打印窗口。提示“已发起，请检查浏览器”，不能写“导出成功”。
- `blocked`：依赖缺失或当前环境不支持，直接显示教程回退。
- `failed`：执行抛错，显示失败原因和教程入口。

只有能够观察到真实产物时才能使用“导出成功”；调用 `click()` 或 `window.print()` 本身不等于文件已经保存。

### 2. 导出策略

优先采用浏览器原生能力：

- HTML：使用 `Blob`、`URL.createObjectURL()` 和带 `download` 属性的链接生成完整报告文件。
- PDF：使用 `window.print()`，按钮文案写成“打印 / 另存为 PDF”，教程明确提示在系统打印窗口中选择“另存为 PDF”。
- 不把浏览器打印包装成“PDF 已生成”。
- 导出前统一调用 `prepareReportForExport()`：
  - 展开全部报告章节；
  - 等待字体、图片和布局稳定；
  - 保证长报告内容完整；
  - 导出结束后恢复原界面状态。
- 打印样式隐藏按钮、悬浮栏、弹窗等非报告控件，并避免粘性控件遮挡内容。

### 3. 依赖缺失时的教程回退

按钮点击后先做能力检查，不能等到异常后仍显示成功 Toast：

```text
报告节点存在
+ 完整报告已生成
+ 对应导出函数/依赖存在
+ 浏览器具备所需原生能力
= 才进入导出流程
```

任一条件失败时：

- 不触发假下载；
- 不显示成功动画或成功 Toast；
- 打开可见的“如何保存报告”教程弹窗；
- 显示具体原因，例如“离线环境未加载 PDF 导出依赖”；
- 提供可操作步骤：
  1. 展开完整报告；
  2. 使用浏览器打印；
  3. 选择“另存为 PDF”；
  4. 或使用 HTML 原生下载。
- 提供“重试导出”按钮，但重试必须重新检查依赖。

教程本身应内嵌在 `Demo/index.html` 或本地静态资源中，保证完全离线可读。官方视频或文档只能作为补充：

- 在线时显示“在浏览器打开官方教程”；
- 离线时明确标注“需要联网”；
- 不把远程 iframe、CDN 脚本或在线视频作为唯一回退；
- 未核验官方来源前不填第三方教程链接。

### 4. 离线资源边界

- 导出演示所需的关键脚本和样式必须本地化；不能运行时依赖 CDN。
- 若某个第三方导出库无法随 Demo 一起提供，则把它视为可选增强，默认降级到 HTML 下载或浏览器打印。
- 图片、字体缺失不能让导出静默成功；应提示哪些资源未包含。
- 资源加载失败后仍需保证教程弹窗、关闭按钮和原生打印路径可用。

### 5. 验收矩阵

| 场景 | 预期结果 |
|---|---|
| HTML 导出能力完整 | 下载内容包含完整报告；提示“已发起下载” |
| PDF 路径可用 | 打开打印窗口；提示选择“另存为 PDF” |
| 导出库缺失 | 不调用导出；显示原因和离线教程 |
| 导出执行抛错 | 显示失败状态、错误摘要、重试和教程 |
| `file://` 离线打开 | 教程可读，HTML/打印原生路径可用 |
| 官方教程无法加载 | 保留本地教程，并提供需要联网的外部入口 |
| 移动端 | 弹窗和按钮不溢出，报告可滚动 |
| 打印预览 | 控件被隐藏，章节不被悬浮栏遮挡，内容完整 |

当前仓库没有实际的 静态 Demo 演示面 `Demo/index.html`、产品文档或对应冒烟测试，因此无法确认现有按钮选择器、导出依赖名称、运行时辅助函数和官方教程地址。最小下一步是提供 static demo surface files，或至少提供 `Demo/index.html` 及相关本地资源和测试；在此之前不应声称方案已经落地或验证通过。

[Ponytail Check] 停在第 1 层 YAGNI：当前只有一次加法调用，没有真实扩展需求，应删除 `ICalculator`、`AbstractCalculator`、`CalculatorFactory` 和 `DefaultCalculator`。

最小实现：

```text
result = a + b
```

若加法包含稳定的业务语义，可保留一个具名函数：

```text
result = add(a, b)
```

只有出现真实需求时才逐级抽象：

- 第二个相同业务规则的使用点：提取普通函数。
- 存在多个真实实现，并需要替换：引入接口。
- 多个实现共享稳定的状态或算法骨架：才考虑抽象基类；优先组合。
- 创建过程复杂，或必须按配置、运行环境选择实现：引入工厂。

“以后可能扩展”、为了单元测试包装纯加法，或只有一个实现，都不足以支撑这些抽象。抽象应由已经发生的变化驱动，而不是预测未来。

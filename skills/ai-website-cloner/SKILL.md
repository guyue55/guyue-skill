# AI Website Cloner

> 一句话钩子：把授权页面变成你本地可维护的 Next.js 代码，带有完整的资产和全局 CSS 提取。

## 你什么时候需要它？
- 看到一个公开页面，想在不复制品牌资产和私有内容的前提下借鉴信息架构。
- 需要快速用 Next.js 复现某个网站的 UI 组件。
- 想要将别人的页面拆解成可复用的代码架构。

## 触发方式
- "帮我克隆这个网站的首页：<url>"
- "完美复刻这个页面的 UI，用到我的项目里"
- "用 ai-website-cloner 抓取并还原这段设计"

## 安全边界与失败回退
- **强制依赖检查**：必须依赖 `JCodesMore/ai-website-cloner-template` 模板。如果当前目录非该模板，强制中断并提示用户先执行 `git clone`。
- **自动化降级 (Fallback)**：核心依赖浏览器自动化 (Browser MCP)。如果无法加载 MCP，必须立即停手，要求用户提供全屏截图与静态 HTML，禁止瞎猜布局。
- **边界**：纯视觉与组件拆解，绝对不涉及后端数据库设计、登录鉴权、复杂状态管理。
- **授权边界**：不得绕过登录、付费墙、反爬或 DRM；不得复制第三方商标、付费素材、私有用户数据或受版权保护的文本内容。

## 核心工作流 (The Foreman Workflow)
1. **支架确认 (Phase 0)**：检查/安装基建模板。
2. **侦察提取 (Phase 1)**：全屏截图(桌面/移动)，提取全局变量(字体/颜色)，记录行为至 `BEHAVIORS.md`。
3. **基建搭建 (Phase 2)**：同步 `globals.css` 和全局资产。
4. **组装派发 (Phase 3)**：按区块生成 `.spec.md` 并**并行派发 Builder Agent**（Worktree模式），验证 `tsc --noEmit`。
5. **视觉 QA (Phase 5)**：并排对比并自动修复偏差。

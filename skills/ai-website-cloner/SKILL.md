# AI Website Cloner

> 一句话钩子：把授权页面变成你本地可维护的 Next.js 代码，带有完整的资产和全局 CSS 提取。

## 你什么时候需要它？
- 看到一个公开页面，想在不复制品牌资产和私有内容的前提下借鉴信息架构、布局节奏和组件关系。
- 需要把自有或已获授权的网站迁移成可维护的 Next.js 代码。
- 想把公开页面拆解成学习用规格说明，而不是发布一个混淆来源的仿冒站。

## 触发方式
- "帮我克隆这个网站的首页：<url>"
- "完美复刻这个页面的 UI，用到我的项目里"
- "用 ai-website-cloner 抓取并还原这段设计"

## 安全边界与失败回退
- **强制依赖检查**：必须依赖 `JCodesMore/ai-website-cloner-template` 模板。如果当前目录非该模板，强制中断并提示用户先执行 `git clone`。
- **自动化降级 (Fallback)**：核心依赖浏览器自动化 (Browser MCP)。如果无法加载 MCP，必须立即停手，要求用户提供全屏截图与静态 HTML，禁止瞎猜布局。
- **边界**：纯视觉与组件拆解，绝对不涉及后端数据库设计、登录鉴权、复杂状态管理。
- **授权边界**：不得绕过登录、付费墙、反爬或 DRM；不得复制第三方商标、品牌识别、付费素材、私有用户数据或受版权保护的文本内容。
- **设计参考边界**：可以学习信息架构、组件拆分、间距节奏、色彩 token、字体层级和交互状态；必须替换品牌资产、专属插图、真实文案、客户 Logo、截图素材和任何会让用户误认来源的元素。
- **用途边界**：适合授权迁移、自有站重建、丢失源码恢复、学习参考和竞品结构分析；不适合钓鱼、仿冒、绕开设计授权、批量抓取或生成混淆来源的网站。

## 核心工作流 (The Foreman Workflow)
1. **支架确认 (Phase 0)**：检查/安装基建模板。
2. **授权确认 (Phase 0.5)**：确认页面是自有、授权或公开允许分析；不清楚时只做学习级结构分析，不做像素级复刻。
3. **侦察提取 (Phase 1)**：全屏截图(桌面/移动)，提取全局变量(字体/颜色)，记录行为至 `BEHAVIORS.md`。
4. **边界脱敏 (Phase 1.5)**：列出必须替换的品牌资产、专属插图、Logo、真实文案、用户数据和受版权保护素材。
5. **基建搭建 (Phase 2)**：同步 `globals.css` 和全局资产，但资产必须来自自有、授权、公开可用或重新生成的安全来源。
6. **组装派发 (Phase 3)**：按区块生成 `.spec.md` 并**并行派发 Builder Agent**（Worktree模式），验证 `tsc --noEmit`。
7. **视觉 QA (Phase 5)**：并排对比结构、比例和状态，不追求复制第三方品牌识别；输出差异和替换证据。

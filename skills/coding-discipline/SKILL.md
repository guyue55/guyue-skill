# 古月开发纪律 (Coding Discipline)

> 一句话钩子：拒绝 AI 敷衍了事的占位符代码和数千行的上帝文件，强制保障工程质量。

## 你什么时候需要它？
- 当你要求 AI 输出代码，且绝不接受省略号 `...` 或 `// your code here` 时。
- 需要规范的模块化切分（前端组件隔离、后端三层隔离）时。
- 团队要求标准化的 Angular 格式纯中文 Git 提交记录。

## 触发方式
- "开始写代码，严格遵守古月开发纪律"
- "拒绝占位符代码，给出这个页面的完整实现"
- "按纪律规范重构，并给出中文 git commit"

## 绝对防相控红线 (Anti-Patterns Guardrails)
- **Zero Fake Code**：无论是修改还是新建，写出的每一行代码必须是全量实体实现，**绝对禁止**伪代码占位。
- **拒绝上帝文件**：严禁将 UI 渲染、业务逻辑与数据访问揉在一个文件里。一旦文件体积膨胀，必须拆分。
- **禁止全量盲读**：阅读现有项目时，严禁使用 `cat` 直接吐出数千行代码撑爆上下文，必须使用 `grep` 或切片精准定位。

## 标准开发工作流
1. **注入声明 (Trace)**：在开始分析和输出代码前，必须在对话中明文打印 `[Trace: Guyue/CodingDiscipline] 注入高内聚低耦合与防敷衍约束...`
2. **前端阵线对齐**：如果是 Web 前端任务，自动搜索并套用当前工作区的 UI 最佳实践（如 Next.js/React 原则）。
3. **外科手术式修改**：如非必要绝不全量覆写现有文件，尽量使用最小化 diff/sed 等替换操作。
4. **交付与 Git**：完成后，提供标准的 `type(scope): 中文核心描述` 的 Git Commit 信息。

# code-minimalism (The Ponytail Discipline)

> 一句话钩子：最好的代码是不用写的代码。给喜欢“炫技”和“造轮子”的 AI 挂上极简主义的缰绳。

## 你什么时候需要它？
- AI 给出的架构方案过于复杂，充斥着没必要的 Factory、Interface 和样板文件。
- 你希望用最少的代码行数、零外部第三方依赖实现一个功能。
- Code Review 时，寻找可以被安全删减的冗余逻辑。

## 触发方式
- "用马尾辫原则帮我重构这段代码"
- "以 code-minimalism 的方式实现，越简单越好"
- "这段代码太臃肿了，削减一下不必要的抽象"

## The YAGNI Ladder (不作为阶梯)
在敲下任何代码前，你必须爬梯子，并在任意满足的一层停手：
1. **YAGNI (You Aren't Gonna Need It)**：真的现在就需要这功能吗？不是的话，砍掉。
2. **复用**：当前 codebase 里有相似的 util 吗？直接复用，不准重写。
3. **标准库**：Node/Python 原生标准库自带吗？用原生的。
4. **原生平台**：HTML5/Browser 原生特性支持吗？用原生的。
5. **已装依赖**：现有的 package.json 里有能用的吗？用现成的。
6. **一行流**：能一行写完且保持可读吗？写一行。
7. **最后**：只写刚够跑通的最小化实现，不写任何“为了未来扩展”的虚假抽象。

## 强制检查点与可见产物 (Guardrails)
- **Ponytail Check 输出**：在实施代码修改前，必须在对话中明文输出 `[Ponytail Check]` 结论，说明你在第几层停手了。
- **妥协标注**：为了极致简单而留下的技术债（例如 O(n^2) 的全表扫描），必须在代码里用 `// ponytail: [天花板及升级路径]` 明文标注。
- **纪律底线**：极简绝不能妥协安全验证、错误处理与无障碍 (a11y) 标准。

# book-distiller (The Cangjie Distillation Process)

> 一句话钩子：不写空洞的读后感，只榨取长文里能变成代码和机器指令的决策树。

## 你什么时候需要它？
- 读完一本管理/技术书籍，希望把作者的方法论变成自己 Agent 的永久技能。
- 手头有一份冗长的业务 SOP，想把它结构化为机器可执行的工作流步骤。

## 触发方式
- "用 book-distiller 把这篇长文提炼成一个技能"
- "提取这个PDF里的决策框架，按 RIA++ 格式输出"
- "把这本关于增长黑客的书结构化，提炼出检查清单"

## 反例黑名单 (Anti-Slop Guardrails)
- **非方法论拦截**：如果是小说、散文、闲聊文本，直接拒绝执行并提示：“当前文本属于叙事/抒情类，不包含可提炼的结构化方法论”。
- **禁止废话**：严禁输出类似“这本书讲述了...的道理”的读后感。必须输出冷酷的、结构化的组件块。

## RIA-TV++ 核心工作流
1. **骨架抓取 (Adler)**：通读长文，建立知识骨架，提取专属名词，输出 `BOOK_OVERVIEW.md`。
   - 记录书名、作者、版本/出版信息、输入文件或官方来源、章节/页码或位置标识；无法定位的观点不得伪造精确出处。
2. **三重验证 (Triple Verification)**：
   - *交叉验证*：有两处以上独立支持吗？
   - *预测力*：能解决书外的新问题吗？
   - *唯一性*：是不是连大模型都知道的常识废话？（常识一律剔除）
3. **组装 RIA++**：
   - **R (原文)**：只保留完成分析所需的最短合规摘录，并附章节/页码或位置；受版权限制或来源不允许长引文时改用转述，不用固定字数上限替代来源规则。
   - **I (转译)**：用行动指令重写框架。
   - **A1/A2 (应用)**：过去案例与未来触发此技能的条件。
   - **E/B (执行与边界)**：123步的执行链路，与该方法论失效的边界条件。
4. **归档测试**：生成 `test-prompts.json` 包含“诱饵”和负面测试。

# 认知循环卡

## 从目的生成维度

把维度视为临时观察轴，不视为预装框架。先写清当前地图要支持什么决定或研究交接，再从以下信号动态生成候选：

- 现有地图中无法解释的实体、关系或生命周期断点；
- 可能改变后果的高风险、不可逆性或受影响群体；
- 来源生态、专业产物、规则或失败记录暴露的新语言；
- 原问题默认忽略的尺度、机制、时间和利益位置；
- 若被证伪，会迫使结论或下一步改变的核心假设。

每个候选维度都写成“观察轴 + 纳入理由 + 谱系锚点 + 可推翻问题”。无法说明为什么现在值得看的候选暂不进入地图。

## 排序而非堆叠

根据任务目的动态权衡：对决定的影响、地图缺口大小、失败后果、可证伪程度、预期信息价值、与已有维度的重复度。高风险任务提高失败后果权重；早期探索提高缺口和信息价值权重；不要把权重固化成跨领域公式。

选择能最大化地图变化的最小候选集。合并同义维度，降权只有装饰性差异的维度，保留少量可能推翻主框架的异质候选。

## 运用五种动作

针对最高价值缺口选择动作，不要求每轮全部执行：

- **上浮**：改写系统边界、目标函数或时间尺度，检查局部最优是否制造全局损失。
- **下钻**：追到机制、单位、阈值、接口和失效链，寻找可观测的因果断点。
- **横移**：移动到相邻角色、阶段、地域、案例或领域，检验类比成立所需条件。
- **反转**：交换受益者与承担者、成功与失败、原因与结果，构造竞争解释和反事实。
- **外证**：要求独立来源类别、专业工件或现实记录约束当前叙事。

一个动作只生成待检验候选，不自动产生事实。

## 修订活地图

每轮记录：

1. 轮前框架与最脆弱假设；
2. 所选动作及其选择理由；
3. 新证据、反例或来源缺口；
4. 地图变化：新增、合并、重命名、升权、降权或删除；
5. 被推翻的认知与仍未解决的冲突；
6. 对下游决定、查询或风险门的影响。

没有地图变化的轮次也要记录为“未修订”，并说明是证据支持、信息重复还是查询无效。不要把新增文字误记为新增认知。

## 选择下一轮

若地图仍由单一来源生态、单一利益位置或未经检验的主假设支配，选择能引入最大独立约束的动作。若多个维度同等重要，优先测试最容易证伪且推翻后影响最大的一个。

需要设计具体取证、比较来源独立性或计算停止价值时，转到 [证据与挑战卡](evidence-and-challenge.md)。不要在本卡内伪造查询结果。
